# General Kaggle Methodology

Reusable operating guidance for Kaggle competitions. Open only the file that
matches the current question.

- `competition_loop.md`: end-to-end competition operating loop.
- `validation_and_leakage.md`: validation design, leakage control, and
  leaderboard alignment.
- `public_research.md`: public notebooks, discussions, claims, and artifacts.
- `reproducibility.md`: candidate identity, records, handoffs, and final
  submission hygiene.

## Append Rules

- Keep this file as the index for general methodology.
- Put durable guidance in topic files, not in this index.
- Generalize before appending: merge near-duplicate lessons into one rule.
- Keep examples anonymous and reusable across competitions.
