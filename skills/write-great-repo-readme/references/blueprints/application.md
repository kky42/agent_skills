# User-Facing Desktop, Mobile, or Web Application

Recommended order:

1. Product name, outcome, and a representative screenshot
2. Try or download links
3. Key capabilities
4. Platform and distribution matrix
5. Minimum system requirements
6. Basic usage
7. Privacy, network, storage, or permission behavior
8. Troubleshooting
9. Development and contribution links
10. License

Separate end-user acquisition from contributor build instructions. When distribution differs by platform, use a compact matrix with explicit minimum versions and caveats.

Keep these boundaries:

- show official downloads before contributor build instructions;
- give screenshots captions, alt text, and a clear product state;
- surface firewall, account, permission, and platform limitations before they matter;
- label nightly, beta, and stable downloads.

**Accepted when:** a supported-platform user can acquire or open the app, complete one basic task, and see relevant permission, privacy, and platform constraints before they matter.
