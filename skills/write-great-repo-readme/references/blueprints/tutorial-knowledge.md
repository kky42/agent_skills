# Tutorial, Curriculum, Roadmap, or Knowledge Repository

Recommended order:

1. What the material teaches or indexes
2. Intended learner and prerequisites
3. What it does not cover
4. How to use the repository
5. Ordered table of contents or category index
6. Suggested paths by goal or available time
7. Progress/checklist conventions
8. Contribution and curation policy
9. License

For long repositories, make the README a navigable map. Use a consistent entry format and categories based on reader goals rather than internal file organization.

Keep these boundaries:

- group links by reader goal;
- define scope for job-readiness or mastery claims;
- place prerequisites before the curriculum;
- use consistent titles, languages, and metadata across entries.

**Accepted when:** the intended learner can confirm prerequisites, choose a goal-based path, reach an observable first milestone, and navigate the remaining material consistently.
