# Hosted Product Plus Package, Component, or SDK

Recommended order:

1. Unified product identity and clear positioning
2. Hosted demo or live product link
3. Split paths: “Use the product” and “Integrate the package”
4. Capabilities of each path
5. Integration quick start
6. Development guide link
7. Documentation and support routing
8. Contributing and license

State precisely what the repository contains. Describe hosted and self-hosted features, limits, and operational responsibilities separately unless their equivalence is verified.

**Accepted when:** each audience can identify what the repository provides, follow its own first-success path, and see the verified differences between hosted and source-available behavior.
