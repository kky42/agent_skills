# Library, SDK, Framework, or CLI

Recommended order:

1. Name and one-sentence capability
2. Focused proof: code/result pair, terminal transcript, or performance evidence
3. Highlights or use cases
4. Install
5. Minimal complete example
6. Expected result
7. Common workflows
8. Requirements and compatibility
9. Configuration or API documentation links
10. Support, contributing, license

Strong pattern: place code directly beside the result it produces. For a CLI, show an actual task rather than opening with the full command reference. For a framework or service, close the loop from file creation to running process to observable response.

Keep these boundaries:

- lead with the first useful command and route the full option list to reference docs;
- ground “fast,” “simple,” and “production-ready” in evidence;
- include imports, initialization, and required configuration in examples;
- keep exhaustive reference documentation outside the README.

**Accepted when:** a fresh reader can install the supported artifact, run the minimal complete example, observe the documented result, and find compatibility limits.
